ISSUE 1: CSV Download Failures
──────────────────────────────
Before: Browser shows "Failed to fetch file"
        CSV never downloads
        Export buttons unreliable

Cause:  Blob URL revoked immediately after click()
        Browser starts download from invalid URL
        Download fails silently

Fix:    Append anchor to DOM
        Click to trigger download
        Remove anchor from DOM
        Delay URL revocation by 5 seconds

Result: CSV downloads reliably
        No more "Failed to fetch file" errors

ISSUE 2: Confusing Import Results UI
────────────────────────────────────
Before: "Import successful! 58 of 57 rows updated"
        Confusing when rows_updated > total_rows
        One CSV email matched multiple DB rows

Cause:  Showed rows_updated / total_rows
        Didn't distinguish CSV rows from DB rows
        User thinks more updated than total

Fix:    Show rows_matched / total_rows as "CSV rows matched"
        Show rows_updated separately if different
        Clear distinction between CSV and DB

Result: "57 of 57 CSV rows matched"
        "58 database rows updated" (shown only if > CSV rows)
        User understands exactly what happened

HOW DOWNLOAD FIX WORKS
================================================================================

Timeline:

  T=0ms:       Blob created, URL generated
  T=1ms:       Anchor element created
  T=2ms:       Anchor appended to DOM
  T=3ms:       Click triggered (download queued)
  T=5ms:       Browser downloads blob data
  T=100ms:     Download manager receives data
  T=200ms:     Browser starts downloading file
  T=5000ms:    Timeout fires, URL revoked
  T=5500ms:    Download completes (most files done by now)

Safety Margin:
  - Most downloads complete within 500-1000ms
  - 5 second timeout provides safe margin
  - URL cleanup guaranteed
  - No browser hangs or memory leaks
