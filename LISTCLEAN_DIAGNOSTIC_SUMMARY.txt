SAFETY GUARANTEES
================================================================================

✅ ADMIN-ONLY
   - Both endpoints require x-admin-token header
   - No public access possible

✅ NO DATA LOSS
   - Only UPDATE operations (never DELETE)
   - All previous history preserved
   - verification_attempt_count still tracked
   - Snapshots captured for audit

✅ REVERSIBLE
   - Reprocess just sets next_retry_at flag
   - Doesn't change core validation data
   - Row can be reset multiple times safely
   - No permanent modifications

✅ STATE-FILTERED
   - Diagnostic/reprocess only affect selected state
   - Can't accidentally hit global unknowns
   - One state at a time by design

✅ QUEUE-DRIVEN
   - Uses existing retry logic
   - Doesn't bypass validation rules
   - Respects 6-attempt limit
   - Follows exponential backoff

✅ EFFORT-LIMITED
   - Sample mode tests 5 rows first
   - Full state requires explicit confirmation
   - Prevents accidental large-scale changes

DATABASE OPERATIONS
================================================================================

DIAGNOSTIC QUERY:
  SELECT id, email_normalized, agency_id, validation_status,
         verification_external_status, verification_bucket_source,
         verification_attempt_count, ...
  FROM other_agency_emails
  WHERE state_slug = 'new-jersey'
    AND validation_status = 'unknown'
  LIMIT 50
  [Indexed on: (validation_status, verification_next_retry_at)]

REPROCESS UPDATE:
  UPDATE other_agency_emails
  SET verification_next_retry_at = NOW(),
      verification_state = 'queued',
      updated_at = NOW()
  WHERE state_slug = 'new-jersey'
    AND validation_status = 'unknown'
    AND verification_attempt_count < 6
    AND id IN (...)
  [Atomic operation, no partial updates]

BEFORE SNAPSHOT: Captured from SELECT results
AFTER SNAPSHOT: Fetched from database post-UPDATE

Result: Audit trail showing exactly what changed

================================================================================
API ENDPOINTS
================================================================================

1. GET /functions/v1/listclean-unknown-diagnostic
   Query: ?state_slug=new-jersey&limit=50
   Auth: x-admin-token required
   Returns: DiagnosticResponse {
     total_unknown_count: number
     sample_count: number
     samples: UnknownEmailRow[]
     analysis: {
       stale_unknown_indicators: number
       potential_real_unknown: number
       has_clean_bucket_source: number
       high_retry_count: number
     }
   }

2. POST /functions/v1/listclean-unknown-reprocess
   Body: {
     mode: "sample" | "full_state"
     state_slug: string
     sample_size?: number
     admin_confirm?: boolean
   }
   Auth: x-admin-token required
   Returns: ReprocessResponse {
     mode: string
     state_slug: string
     rows_reset: number
     before_snapshots: array
     after_snapshots: array
     queue_processor_kicked: boolean
   }
