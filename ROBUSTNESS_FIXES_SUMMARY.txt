FIX 1: NAME COLUMN INDEX-0 BUG (CRITICAL)
──────────────────────────────────────────
Issue:
  Excel files where Name is the first column (index 0) would fail to import
  Error: "No 'Name' or similar column found in Excel sheet"

Root Cause:
  Line 134 in src/utils/excelImportParser.ts used falsy check:
    if (!mappedFields['name'])  ← Fails when index is 0 (falsy)

Fix Applied:
  Changed to explicit undefined check:
    if (mappedFields['name'] === undefined)  ← Only true if undefined

Impact:
  ✅ Excel files with Name in column A now work
  ✅ All other column positions unaffected
  ✅ High-likelihood bug (natural column ordering)

File Changed:
  src/utils/excelImportParser.ts (line 134)

================================================================================

FIX 2: EXCEL ROWS SENT AS JSON (ROBUSTNESS ENHANCEMENT)
──────────────────────────────────────────────────────

Issue:
  Excel rows were parsed on client, converted to CSV, then sent to backend
  CSV conversion required complex escaping (semicolons, quotes, newlines)
  Fragile edge cases in CSV parsing could break imports

Solution:
  Send Excel-parsed rows directly as JSON to backend
  - Direct object serialization (foolproof)
  - Backend already supports JSON import mode
  - Same validation and results as before

Changes:

  File: src/utils/excelImportParser.ts
    - REMOVED: convertParsedRowsToCSV() function (complex CSV escaping)
    - ADDED: convertParsedRowsToJSON(rows) → JSON.stringify(rows)

  File: src/pages/AdminOtherAgenciesImportPage.tsx
    - CHANGED: Import statement (CSV → JSON converter)
    - CHANGED: handleImport() function:
        OLD: fileContent = convertParsedRowsToCSV(parsedRows); importMode = 'csv';
        NEW: fileContent = convertParsedRowsToJSON(parsedRows); importMode = 'json';

Flow After Fix:
  Excel Upload
    ↓ (client-side)
  Parse XLSX → Array of objects
    ↓
  Serialize to JSON string
    ↓
  Send to other-agencies-import-v2 with mode: "json"
    ↓ (backend - unchanged)
  Parse JSON rows
    ↓
  Apply standard import logic
    ↓
  Create/update agencies

Benefits:
  ✅ No CSV parsing edge cases
  ✅ Simpler, cleaner data flow
  ✅ Better error handling
  ✅ Scales reliably to 1000+ rows
  ✅ Uses proven backend JSON logic

Impact:
  ✅ More robust import path
  ✅ Same results as before
  ✅ Better scaling at size
